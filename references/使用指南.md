# DeepGarden 公司研报精读 v1.1 — 安装者使用指南

> 适用对象：想用本 Skill 把券商公司研报拆成"可复核、可追溯、可累积跟踪"Obsidian 笔记的投资者。
> 相对 v1.0 的核心升级：**按公司归档 + 研报跟踪对比 Hub（Step5）+ 与定期报告系列双向印证**。
> 配套文件：`SKILL.md`（框架）、`scripts/extract_report.py`（PDF 抽文本）、
> `references/example-output-uimages.md`（真实精读样例）、`references/example-hub-uimages.md`（真实 Hub 样例）、`references/跟踪hub模板.md`（Hub 骨架）。

## 一、开始前准备

1. **环境**：WorkBuddy 管理的 Python（终端直接 `python`）。脚本依赖 `pdfplumber`，首次运行自动安装；失败则手动 `pip install pdfplumber`。
2. **输入**：1 个研报 PDF（深度报告 / 个股点评 / 首次覆盖均可）。建议文件名含「日期-券商-公司-代码-标题」，例如：
   `20260707-华创证券-联影医疗-688271-深度研究报告系列二：从国产替代到全球竞争.pdf`
3. **两个系列已就位**：本 Skill（研报精读）与 `deepgarden-earnings-tracking`（定期报告）在仓库根并列。
   若尚无定期报告系列，Hub 仍会按研报维度建立，但"与定期报告印证"会标注"待补充"。

## 二、目录架构（按公司累积）

```
研报精读/
├─ 00-研报精读总索引.md          # 所有公司、所有研报的顶层索引（自动维护）
└─ <公司名>/
    ├─ 00-<公司名>-研报跟踪.md   # ★ 跟踪对比 Hub（Step5 生成/更新）
    ├─ 原始报告/                  # COPY 的研报 PDF（不移动用户原件）
    └─ 精读/                      # 每篇研报一份 MD
```

## 三、推荐运行流程（对话式）

直接对话触发即可，例如：

> "用 DeepGarden 研报精读 框架，帮我精读 `<你的研报PDF路径>/20260707-华创证券-联影医疗-688271-...pdf`，并跟踪对比"

Skill 会自动执行：

1. **抽文本**：`scripts/extract_report.py` 把 PDF 转成带 `[Pxx]` 标记的 txt（存 `_extract/` 临时目录）。
2. **按公司归档**：识别公司名 → 建/更新 `研报精读/<公司名>/`，COPY 源 PDF 进 `原始报告/`。
3. **Step1 标题拆解 + 盈利预测变化 + 趋势判断**：标题逐词映射到投资逻辑；找"前值 vs 现预测"；提炼成长框架与评级。
4. **Step2 全网核实 + 独立复核**：WebSearch 交叉验证营收/净利/海外/份额；主动找"增速断层、毛利背离、现金流背离、跨研究分歧、数据口径局限"等预期差。
5. **Step3 近半年同公司研报趋势**：**双源比对**——先读本公司的 `00-<公司>-研报跟踪.md`（本地历史），再做 Web 检索（同花顺/东财 F10、富途），汇总机构评级/目标价/净利预测，对比共识与本研报。
6. **Step4 估值分位 + PEG + 走势区间 + 风险**：取 PE/PB/PS 5 年分位（金融数据 skill 优先，否则 WebSearch 雪球/亿牛并标注矛盾）；算 PEG；给保守/基准/乐观三档区间；结合资金面与基准指数给风险。
7. **★ Step5 研报跟踪对比 Hub + 跨系列印证**：生成/更新 `00-<公司>-研报跟踪.md`——盈利预测修正轨迹、评级/目标价轨迹、**与定期报告逐项印证（✓/⚠️/✗）**、共识温度计、滚动待验证点；并在每篇精读 MD 内新增「与定期报告印证」段，双向 wikilink 闭环。
8. **成稿 + 归档**：写出 `研报精读/<公司>/精读/<公司>-<代码>-<券商>-<YYYYMMDD>-精读.md`；更新公司 Hub 与顶层总索引。

## 四、预期输出

- 每篇精读 MD：frontmatter（含 `periodic_link` / `hub_link`）+ 摘要(TL;DR) → 一/二/三/四步 → **五、与定期报告印证** → 免责声明。所有关键论断带 `[Pxx]` 引用。
- 公司 Hub（持续累积）：把零散研报变成可追踪资产，跨期预期差与兑现度一目了然。
- 两系列互证：研报精读 ↔ 定期报告，wikilink 双向闭环。

## 五、常见坑

- **PDF 是扫描件/无文字层**：`pdfplumber` 抽不到文本。需用 OCR（markitdown-skill 或本地 OCR）先转，再喂 txt。
- **估值分位站点矛盾**：雪球与亿牛网(eniu)可能给出相反分位，以逻辑自洽者为准并显式提示矛盾，不盲信单一来源。
- **"前值 vs 现预测"找不到**：点评类研报常不写前值。改为在 Step3 用近半年共识变化推断调整方向，Hub 轨迹以"共识趋势"补充。
- **Hub 需持续维护**：单篇精读价值有限，价值在累积。每新增一篇务必刷新 Hub；定期报告出新后，回看 Hub 的「印证」结论是否被裁决（✓/⚠️/✗ 更新）。
- **勿删用户原件**：归档只 COPY，源 PDF 留在原位。
- **非投资建议**：文末免责声明为强制项。

## 六、预检清单

- [ ] PDF 路径正确、可读取
- [ ] 输出根目录已确定（默认 `研报精读/`）
- [ ] 同公司是否已有 `研报精读/<公司名>/`（决定新建还是更新 Hub）
- [ ] 是否已有 `定期报告整理/<公司名>/`（决定能否做跨系列印证）
- [ ] 网络可用（WebSearch / F10 站点）
- [ ] 已知晓：成稿含独立复核与跨系列印证观点，需自行判断
