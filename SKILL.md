# DeepGarden 公司研报精读 Skill（v1.1）

> 本 Skill 属于 **DeepGarden 投资研究系列**，与 `deepgarden-ipo-research`（DeepGarden 新股研究）、
> `deepgarden-earnings-tracking`（DeepGarden 定期财报跟踪）并列。
> **DeepGarden 系列投资 Skill 知识库分享由 Deep Garden 出品** —— 欢迎下载、讨论；后续将持续发布相关 Skill。

对单篇公司深度/个股研报进行**四步结构化精读**，并进一步把同一公司的多篇研报**持续累积、跟踪对比**，
与 `deepgarden-earnings-tracking` 产出的**定期报告系列**形成**双向印证**的两大研究系列。

- **四步精读**：标题拆解 / 盈利预测变化 → 业务数据核实与独立复核 → 近半年同公司研报趋势 → 关注重点+估值分位+PEG+走势区间+风险。
- **第五步（v1.1 新增）**：生成/更新「**研报跟踪对比 Hub**」——按公司聚合所有研报的盈利预测修正轨迹、评级/目标价轨迹，并**逐项与定期报告实际业绩交叉印证**，输出共识温度计与滚动待验证点。
- 核心不是"复述研报"，而是**对研报做独立复核、跨期跟踪与跨系列印证**，帮投资者识别预期差、验证兑现度、降低被单一卖方叙事带偏的概率。

## When to use
- 用户丢来一篇券商公司研报（深度报告 / 个股点评 / 首次覆盖），要求"精读""拆解""复核""跟踪"。
- 用户想了解：这份研报核心逻辑、盈利预测变了什么、数据是否靠谱、其他机构怎么看、现在贵不贵。
- 用户希望**把关注公司的研报持续放在一起**，对比不同券商/不同时点的预测演变，并**与定期报告互相验证**。
- 输出目标为 Obsidian 仓库 / Markdown 笔记，强调结构化、可追溯（页码级 `[Pxx]` 引用）与可累积（按公司归档）。

## 两大研究系列与目录架构（关键）

本 Skill 与 `deepgarden-earnings-tracking` 在仓库根形成**并列双系列**，互为印证：

```
<仓库根>/
├─ 定期报告整理/            ← 系列一：实际业绩（deepgarden-earnings-tracking 产出）
│   └─ <公司名>/
│       ├─ 00-<公司名>索引.md        # 跨年财务快照 + 各期阅读整理链接
│       ├─ 原始报告/                  # COPY 的 PDF
│       └─ 阅读整理/                  # 每期一份 MD
│
└─ 研报精读/                ← 系列二：卖方预测/判断（本 Skill 产出）
    ├─ 00-研报精读总索引.md          # 所有公司、所有研报的顶层索引
    └─ <公司名>/
        ├─ 00-<公司名>-研报跟踪.md   # ★ 跟踪对比 Hub（Step5 生成/更新）
        ├─ 原始报告/                  # COPY 的研报 PDF（不移动用户原件）
        └─ 精读/                      # 每篇研报一份 MD
```

- **相对路径约定**（两系列同处仓库根）：
  - 研报精读 → 定期报告：`../../定期报告整理/<公司名>/00-<公司名>索引.md`
  - 定期报告 → 研报精读：`../../研报精读/<公司名>/00-<公司名>-研报跟踪.md`
- 用 Obsidian `[[wikilink]]` 双向互链，保证两系列"你中有我"。

## 输入
- 1 个研报 PDF（或用户已抽取的 txt）。支持 A股/港股/美股个股深度研报。
- 可选：用户指定输出根目录（默认 `研报精读/`）；公司名从文件名/研报封面自动识别。

## 四步框架 + 第五步 总览

```
Step1  标题关键词拆解 + 盈利预测变化重点解读 + 个股趋势判断（基于研报本身）
Step2  业务数据全网核实 + 独立复核判断（找预期差/风险，不盲从）
Step3  近半年同公司研报 / 出具机构 / 盈利预测变化趋势（横向比对，含本地 Hub 历史）
Step4  关注重点+时间节点 + PE/PB/PS 5年分位 + PEG + 走势区间 + 市场情绪/风险 + 免责声明
Step5  ★ 研报跟踪对比 Hub：盈利预测修正轨迹 + 评级/目标价轨迹 + 与定期报告印证 + 共识温度计 + 滚动验证点
```

## Workflow

### 0. 抽取文本（带页码标记）
若输入是 PDF，用 `scripts/extract_report.py`（依赖 `pdfplumber`）抽取为带 `[Pxx]` 标记的纯文本：
```bash
python scripts/extract_report.py --src "<研报.pdf>" --out "<工作目录/_extract>"
```
- 每个 PDF 页面前缀 `[Pxx]`，便于分析时用 Grep 定位行号、用 Read 读取局部，避免一次性灌入大文本。
- 若用户已给 txt，直接用，跳过本步。
- 运行环境：使用 WorkBuddy 管理的 Python；`pdfplumber` 缺失时脚本会尝试自动 `pip install`。

### 1. 归档（按公司，COPY 不移动）
识别公司名后，**先建/更新 per-company 目录**：
- `研报精读/<公司名>/精读/` 放精读 MD；
- `研报精读/<公司名>/原始报告/` **COPY** 源 PDF（保留用户原件，绝不移动/删除）；
- `研报精读/<公司名>/00-<公司名>-研报跟踪.md` 为跟踪 Hub（Step5 维护）。
- 用 Grep 在抽取文本中定位关键章节：封面/财务摘要、投资逻辑、盈利预测与估值、风险提示、附录财务预测表。
- 重点抓取：评级、目标价、当前价、分析师、盈利预测表（含"前值/现预测"对比）、成长曲线论述、估值方法。

### 2. Step1 — 标题关键词拆解 + 盈利预测变化 + 趋势判断
- **标题关键词拆解**：把研报标题逐词映射到投资逻辑（例："从国产替代到全球竞争"→增长引擎切换；"重新审视"→预期差；"系列二"→跟踪升级）。用表格呈现「关键词 → 隐含逻辑 → 研报对应论证（带 [Pxx]）」。
- **盈利预测变化重点解读**：在盈利预测/估值章节找"前值 vs 现预测"（多数深度报告会写明，如"26-28年前值为23.1/28.8/35.3亿元，现预测23.1/28.8/36.1亿元"）。列表对比 + 解读上调/下调的年度与幅度、隐含假设、目标价推导（PE倍数×EPS）。**若点评类研报未写前值，跳过本对比、改在 Step3 用近半年共识变化推断调整方向。**
- **个股趋势判断（基于研报）**：提炼研报的核心成长框架（如"五条成长曲线"），说明评级与未来 6 个月相对基准指数的超额收益区间（参考研报末尾评级体系）。

### 3. Step2 — 业务数据全网核实 + 独立复核（关键，体现"独立"）
- **核实**：对研报披露的关键业务/财务数据（营收、净利、海外收入、份额、装机量、服务收入等），用 WebSearch 检索公司财报/公告/权威媒体（中证报、界面、同花顺 F10、东方财富 F10 等）交叉验证，标注 ✓一致 / △采信但有口径局限 / ⚠️存疑。
- **独立复核（找预期差）**：主动发现研报未充分提示的风险，至少覆盖：
  1. **近期增速断层**：如最新季报增速 vs 全年指引（例：Q1 净利 +7.7% 但全年指引 +23.4%，H2 需强反弹）。
  2. **利润率假设**：研报毛利率假设 vs 实际近期走势是否背离。
  3. **现金流与净利背离**：用附录现金流量表检查经营现金流/每股经营现金流是否大幅低于净利（增长质量）。
  4. **资产负债占用**：存货、应收是否随收入高增（渠道铺货/回款风险）。
  5. **与本地/其他独立研究分歧**：若用户仓库内有同公司估值/研究文档（优先查 `DeepGarden/投资知识库/03-公司研究/`、 `研报精读/<公司名>/`），主动比对结论方向。
  6. **数据口径局限**：第三方招标份额（医招采/医工研习社）为"按数量"小样本，标注参考而非精确值。
- 每条复核发现标注 ⚠️ 并给出可验证的观测点（这些观测点会进入 Step5 的"滚动待验证点"）。

### 4. Step3 — 近半年同公司研报 / 机构 / 盈利预测趋势
- **双源比对**（v1.1 强化）：
  - **本地 Hub 历史**：先读 `研报精读/<公司名>/00-<公司名>-研报跟踪.md`（若存在），把本研报的盈利预测/评级/目标价与**过去本地已精读的同公司研报**做直接对比，得到"本仓已记录的修正轨迹"。
  - **Web 新鲜检索**：WebSearch 关键词如「<公司> <代码> 研报 2026 盈利预测 机构」「<公司> 目标价 评级」。优先同花顺 F10 / 东方财富 F10 盈利预测页（含近六月机构数、EPS/净利均值、各机构明细）、富途牛牛（目标价分布）。
- **汇总表**：机构 / 日期 / 评级 / 目标价 / 当年净利预测。
- **共识 vs 本研报**：对比覆盖机构数、净利预测均值与本研报差异（谁高谁低、偏离幅度）；目标价区间（最高/最低/均值）与本研报位置（上限/下限/居中）。
- **趋势建议**：盈利预测近半年上调/下调方向、长期（2-3年后）预期分歧大小、本研报态度（上调/维持/下调）。给出"预期差在哪"的定性结论。

### 5. Step4 — 关注重点 + 估值分位 + PEG + 走势区间 + 风险
- **关注重点与时间节点**：列表给出未来 1-4 个季度需跟踪的催化/验证点（中报、政策招标节奏、海外获批、高端产品放量、服务收入占比、激励计划等）。**这些点同步写入 Step5 Hub 的"滚动待验证点"。**
- **PE/PB/PS 五年分位**：
  - 优先调用金融数据 skill（westockdata / a-stock-data / neodata-financial-search / wb-finance-skill）取历史估值分位；
  - 若无，用 WebSearch（雪球「估值分位」长文、亿牛网 eniu.com/gu/<code>）获取，并**明确标注数据口径与日期**；
  - ⚠️ 不同站点分位可能矛盾（如 eniu 与雪球），须交叉比对、以逻辑自洽者为准，并提示矛盾。
  - 若只能拿到当前 PE/PB/PS 与历史区间（最高/最低/均值），自行估算分位区间并标注"估算"。
- **PEG**：用报告或共识的 远期 PE ÷ 盈利增速 计算（分别给 当年/次年/第三年），说明短期与长期的 PEG 差异。
- **走势区间建议**：分保守/基准/乐观三档，结合（a）估值分位支撑、（b）盈利预测、（c）政策/情绪，给出价格区间（非承诺，标注"独立判断"）。
- **市场情绪 / 股指 / 风险**：
  - 标明研报基准指数（多为沪深300），简述近 12 月相对表现；
  - 用 WebSearch 取最新价、成交额、融资余额分位、主力/游资/散户资金流向，判断短期情绪；
  - 列出风险（结合研报风险提示 + Step2 独立发现），用 🔴高 / 🟠中 标注等级。
- **免责声明**：文末固定附「仅供参考研究，不构成买卖建议」+ 数据来源 + 页码引用说明。

### 6. ★ Step5 — 研报跟踪对比 Hub + 与定期报告印证（v1.1 核心新增）
每产出一篇精读后，**必须生成或更新** `研报精读/<公司名>/00-<公司名>-研报跟踪.md`。这是把零散研报变成"可跟踪研究资产"的关键。

**6.1 先读取已有的同公司资料**
- 若 `研报精读/<公司名>/00-<公司名>-研报跟踪.md` 已存在 → 本次为**更新**：追加新研报行、刷新轨迹与印证。
- 读取 `../../定期报告整理/<公司名>/00-<公司名>索引.md`（定期报告系列的跨年快照），作为"实际业绩锚"。
- 若定期报告系列尚不存在，注明"（定期报告系列未建立，待补充印证）"，但 Hub 仍按研报维度建立。

**6.2 Hub 内容结构（模板见 `references/跟踪hub模板.md`）**
1. **研报清单（按时间倒序）**：精读日期 / 券商 / 标题 / 评级 / 目标价 / 链接。
2. **盈利预测修正轨迹**：报告日期 / 券商 / 当年净利E / 次年净利E / 第三年净利E / 相对前次修正 / 来源。→ 一眼看出"市场在上调还是下调预期"。
3. **评级 / 目标价轨迹**：报告日期 / 券商 / 评级 / 目标价 / 相对前次。
4. **★ 与定期报告印证（核心）**：逐条把研报的**关键假设/预测**与定期报告**实际业绩**对照，结论用 ✓兑现 / ⚠️待验 / ✗落空：
   - 例：研报"2026E 净利 +23.4%" vs 定期报告"2026Q1 净利 +7.7%" → ⚠️ H2 需强反弹方能兑现；
   - 例：研报"毛利率 2028E 升至 49.8%" vs 定期报告"2025 47.0%、Q1 47.2%(-2.8pct)" → ⚠️ 待验；
   - 例：研报"2025 营收 138 亿" vs 定期报告"2025 营收 138 亿(+34%)" → ✓ 一致。
   - 每条给出来源 wikilink（研报精读 MD 的 [Pxx] 段落 + 定期报告 MD 的具体页/快照）。
5. **共识温度计**：覆盖机构数、净利预测均值、本仓各研报相对共识位置、长期（2-3年）分歧大小、近期修正方向。定性给出"上修/下修/分歧扩大"。
6. **滚动待验证点**：从各篇精读 Step2/Step4 抽取的观测点汇总（H2 反弹、毛利率拐点、现金流转化、海外占比、服务收入占比、政策/招标节奏、激励解锁等），标注提出研报与时间。

**6.3 双向印证落点**
- Hub 内用 `[[../../定期报告整理/<公司名>/00-<公司名>索引]]` 链接到定期报告系列；
- 定期报告系列的 `00-<公司名>索引.md` 应反向加一行链接到本 Hub（见 `deepgarden-earnings-tracking` 的互证说明），形成闭环。

### 7. 输出
- 精读 MD 文件名：`<公司>-<代码>-<券商>-<YYYYMMDD>-精读.md`，写入 `研报精读/<公司名>/精读/`。
- 每篇精读 MD 必须包含：
  - frontmatter 增加 `periodic_link`（指向定期报告索引）与 `hub_link`（指向本 Hub）；
  - 正文末（免责声明前）新增 **「## 与定期报告印证」** 小节：用 3–6 条把本研报关键假设 vs 最近一期定期报告实际对照，结论 ✓/⚠️/✗；并链接到 Hub。
- 维护/更新 `研报精读/<公司名>/00-<公司名>-研报跟踪.md`（Step5）。
- 维护/更新顶层 `研报精读/00-研报精读总索引.md`（公司、券商、日期、评级、目标价、链接 + Hub 链接）。

## 输出模板（节选骨架）

## Notes / pitfalls
- **独立复核是第一价值**：不要只复述研报。Step2 的"增速断层 / 现金流背离 / 跨研究分歧"才是精读精髓。
- **跟踪 Hub 是累积价值**：v1.1 起，单篇精读必须沉淀进 Hub。没有 Hub，研报就是一次性消费；有了 Hub，跨期预期差与兑现度才可被持续追踪。
- **双向印证要闭环**：研报精读 MD 指向定期报告索引 + Hub；Hub 也指向定期报告；定期报告索引反向指向 Hub。三处 wikilink 缺一不可。
- **页码引用必须真实**：[Pxx] 来自抽取文本的行标记，禁止编造页码。
- **估值分位口径**：不同站点（雪球/亿牛）可能给出矛盾分位，必须交叉验证并提示；无法实时取数时标注"估算"。
- **PEG 用远期口径**：用预测 PE ÷ 预测盈利增速，分别给当年/次年/第三年，避免单点误判。
- **COPY 不移动**：源 PDF 归档时复制，绝不删除用户原件。
- **非投资建议**：文末免责声明为强制项，不可省略。
